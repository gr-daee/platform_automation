# Test Cases Documentation

## Overview
Test case documentation organized by automation and manual testing approaches.

## Structure

### Automation (`automation/`)
Documentation for automated test cases organized by module.

**Current Modules:**
- `auth/` - Authentication test cases
- `o2c/` - Order-to-Cash test cases (pending)

### Manual (`manual/`)
References to manual test case documentation.

## Test Case Documentation Standards

### Required Information
- Test scenario description
- Gherkin syntax (for automated tests)
- Coverage details
- Status (✅ Automated | ⏳ Pending | ❌ Failed)
- Last updated date
- Related files (feature files, POMs, step definitions)

### Documentation Location
- **Module Test Cases**: `../modules/[module]/test-cases.md` - Test case inventory
- **Detailed Documentation**: `automation/[module]/` - Detailed test documentation
