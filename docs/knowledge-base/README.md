## Purpose

This knowledge base contains information that spans multiple modules:
- Database schemas used across modules
- Business rules that affect multiple areas
- System architecture and design decisions
- API contracts and endpoints

## When to Add Here

Add to knowledge-base when information:
- ✅ Affects multiple modules
- ✅ Is system-wide (architecture, database)
- ✅ Is shared across teams
- ✅ Needs to be discoverable by all

## When to Add to Module Docs

Add to `modules/[module]/knowledge.md` when information:
- ✅ Is specific to one module
- ✅ Is module implementation details
- ✅ Is component-specific
